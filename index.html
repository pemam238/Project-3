Visualization

Design Rationale

This version uses a North Pole-centered azimuthal equal-area projection, but represents each grid cell as a lightweight polygon instead of a full-resolution raster or SVG-heavy map. This preserves Arctic spatial patterns while keeping the page performant and interactive in the browser.

Data Transformations

Data is loaded from arctic_data.csv. Longitudes above 180° are converted into the −180° to 180° range for D3 projection compatibility.

Development Process

I first attempted to reproduce the original Arctic map using individual SVG polygons for every climate grid cell. However, the full-resolution dataset created too many projected paths and significantly slowed browser rendering.

To improve performance, I spatially aggregated nearby grid cells and rendered them as lightweight polygons with D3.js. Additional interactivity was added through a metric toggle, tooltip hover interactions, dynamic legends, and a brushable histogram filter.
