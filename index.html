Design Rationale

A polar azimuthal equal-area projection centered on the North Pole mirrors the Cartopy NorthPolarStereo view. Discrete color bins (YlOrRd for heat days, RdBu_r for anomalies) match the matplotlib normalization. Brushable histograms cross-filter both maps so you can isolate where high heat days co-occur with large positive anomalies.

Data Transformations

Data is loaded from arctic_data.csv (same directory as this HTML file). Longitudes > 180° are converted to negative (−180 to 180) for the D3 projection. Generate the CSV from your xarray dataset with: 

heat_df  = heatwave_days.to_dataframe(name='heatDays').reset_index()
extra_df = extra_days.to_dataframe(name='extraDays').reset_index()
df = heat_df.merge(extra_df, on=['lat','lon'])
df[['lat','lon','heatDays','extraDays']].dropna().to_csv('arctic_data.csv', index=False)

Development Process

Development began by replicating the Cartopy projection in D3 using d3.geoAzimuthalEqualArea rotated to the North Pole, then matching bin boundaries and color ramps from the Python code. Brushing was added using d3.brushX on histogram panels that drive opacity on both map layers simultaneously.
