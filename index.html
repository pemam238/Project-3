Design Rationale

This version uses a North Pole-centered azimuthal equal-area projection, but represents each grid cell as a lightweight circle instead of a full SVG polygon. This keeps the Arctic spatial pattern while making the page much faster to load.

Data Transformations

Data is loaded from arctic_data.csv. Longitudes above 180° are converted to the −180 to 180 range for D3 projection. 

heat_df  = heatwave_days.to_dataframe(name='heatDays').reset_index()
extra_df = extra_days.to_dataframe(name='extraDays').reset_index()
df = heat_df.merge(extra_df, on=['lat','lon'])
df[['lat','lon','heatDays','extraDays']].dropna().to_csv('arctic_data.csv', index=False)

Development Process

I first attempted to reproduce the original map using SVG grid polygons, but the full dataset created too many projected paths in the browser. I simplified the design by plotting projected grid points as circles, then added a metric toggle, colorbar, tooltip, and brush-based filtering with D3.
